# GitHub Copilot クレジット消費量計算ツール

GitHub Copilot のクレジット消費量を事前に見積もるためのスタンドアロン Web ツールです。  
GitHub Copilot は 2026/6/1 に premium request 方式の課金を廃止し、トークンベースの従量課金（GitHub AI Credits, 1 credit = $0.01）に移行しました。
プロンプト・添付ファイル・使用モデルを入力するだけで、**Ask モード / Plan モード / Agent モード** の3モードについて、1リクエストあたりのクレジット消費量を算出し、モデル間の比較も行えます。Plan/Agent は内部で複数回のモデル呼び出しが起こる前提の反復シミュレーションで概算します。

> チーム単位の予算管理・プランティア適合判定、Copilot cloud agent（GitHub Actions minutes 消費含む）はスコープ外です。

---

## エンドユーザー向け: 使い方

### 1. ツールを開く

`dist/index.html` をダブルクリックしてブラウザで開く。  
インターネット接続・インストール不要。

### 2. 入力する

**モードを選択:** Ask / Plan / Agent（セグメントコントロールで切り替え）

**共通項目:**

| 項目 | 説明 |
|---|---|
| 使用モデル | 登録済みモデルから使用予定のものを選択（provider別にグルーピング表示） |
| プロンプト・指示文 | 実際に送信するプロンプトをそのまま貼り付ける |
| 参照コード・設計書（テキスト） | コピー&ペーストするコードや設計書テキスト |
| 参照ファイル・フォルダ | ファイル種別とサイズ(KB)を入力（複数可、参考値。ソースコード/フォルダにも対応。精度を上げたい場合は本文をテキスト欄へ貼り付け） |

**Ask モード固有:**

| 項目 | 説明 |
|---|---|
| 何回目のやり取りか | 今回が同一スレッドの何ターン目かを入力する。ターン数が履歴分の入力トークンとキャッシュ割引率の両方を自動的に決める |
| 想定回答文字数 | 生成される回答の概算文字数 |

**Plan / Agent モード固有:**

| 項目 | 説明 |
|---|---|
| タスク規模 | 軽微 / 中規模 / 大規模から選択。反復回数・増分・出力・サブエージェント数のプリセットが自動適用される |
| 詳細調整（任意） | 反復回数・1反復あたりの増分/出力・最終成果物の出力・サブエージェント数を折りたたみ UI で個別に上書き可能 |

※ system prompt・tool definitions 等の見えない文脈分は、モードごとの概算オーバーヘッドとして自動加算されます。
※ Ask のキャッシュ割合は利用者が数値を直接入力するのではなく、「何回目のやり取りか」から自動推定されます。Plan/Agent は反復シミュレーションによりキャッシュ書き込み・読み込みが自動計算されます。

### 3. 結果を確認する

- **消費クレジット・概算金額 (USD)**: 1リクエストあたりの消費量
- **内訳**: 入力 / キャッシュ読み込み / キャッシュ書き込み / 出力ごとのクレジット
- **モデル別比較表**: 登録済みモデルすべてのコストを昇順で比較（全モード共通）

### 4. 事務局申請に活用する

計算結果を「事前見積もり提出フォーマット」に記載し、事務局に申請してください。

## 計算ロジックの概要

**Ask モード**（1回のモデル呼び出し）:

```
消費クレジット = [(Input × Input単価) + (Cached Input × Cached単価) + (Output × Output単価)] ÷ 10,000
```

「何回目のやり取りか」が履歴トークン量とキャッシュ割引率の両方を決定します（T=1→0%, T=2→25%, T=3〜4→35%, T≥5→50%）。

**Plan / Agent モード**（反復シミュレーション、閉形式）:

```
cacheWriteTokens  = C0 + (N-1)・G
cachedInputTokens = (N-1)・C0 + G・(N-1)(N-2)/2
outputTokens      = N・O + F
```

- `C0`: 初回送信コンテキスト（モードオーバーヘッド＋プロンプト＋参照情報）
- `N`: 反復回数（モデル呼び出し回数） / `G`: 1反復あたりの増分トークン
- `O`: 1反復あたりの出力トークン / `F`: 最終成果物の出力トークン

サブエージェントは1体ごとに同エンジンの小型ループとして計算し合算します。ロングコンテキスト単価（GPT-5.4/GPT-5.5/Gemini 3.1 Pro）の判定は、実際の課金がリクエスト単位のため、反復シミュレーション全体の合計ではなく「ループ中で最大となる1回の呼び出し分のコンテキスト量」で行います。詳細は `docs/説明資料/計算ロジック説明.md` を参照してください。

この計算モデルはGitHub公式の固定値ではなく、事前見積もり用の保守的な仮定です。
実際の消費量は、モデル・セッション継続状況・参照情報の一致度・ツール設定変更などにより変動します。

## 代表シナリオ（試してみる）

実際の数値（クレジット・USD）は `src/rates.js` の単価改定に応じて変わるため、本READMEには記載しません。
下記の入力例をそのままツールに入力し、実際の画面で結果を確認してください。

### シナリオ1: Ask でユニットテスト自動作成を依頼する

1. モード: **Ask**
2. 使用モデル: **Claude Sonnet 5**（例。実際に使う予定のモデルに置き換えてよい）
3. プロンプト・指示文: 「このモジュールのユニットテストを作成してください。エッジケースも含めてください。」
4. 参照コード・設計書（テキスト）: テスト対象のソースファイル本文を貼り付ける、または参照ファイルとして Markdown/テキスト 20〜50KB 程度を追加
5. 何回目のやり取りか: **1**（新規スレッドを想定）
6. 想定回答文字数: 生成されるテストコードの概算文字数（例: 1500）
7. 「クレジット消費量を計算」をクリックし、消費クレジット・USD・内訳・適用キャッシュ率・モデル比較表を確認する

### シナリオ2: Agent で中規模の実装タスクを依頼する

1. モード: **Agent**
2. 使用モデル: **Claude Sonnet 5**（例。実際に使う予定のモデルに置き換えてよい）
3. プロンプト・指示文: 「この機能を実装し、テストも追加してください。」
4. 参照コード・設計書（テキスト）または参照ファイル・フォルダ（ソースコード種別）を追加
5. タスク規模: **中規模**
6. 詳細調整は変更せずプリセットのまま計算する
7. 「クレジット消費量を計算」をクリックし、消費クレジット・内訳（入力/キャッシュ読み/キャッシュ書き/出力）・反復回数の前提を確認する。実際の消費量とは差が出る可能性がある点に注意する

## クレジット単価の更新方法

GitHub Copilot の料金体系が改訂された場合は以下の手順で更新する。

1. [GitHub Copilot 公式ドキュメント](https://docs.github.com/copilot/managing-copilot/monitoring-usage-and-entitlements) で最新単価を確認
2. `src/rates.js` を編集
   - `MODEL_RATES`: モデル別クレジット単価（4フィールド: input / cachedInput / cacheWrite / output）
   - `FEATURE_OVERHEAD_TOKENS`: Ask/Plan/Agent の見えない文脈分オーバーヘッド（実測較正が必要な暫定値）
   - `EXPERIMENTAL_AGENTIC_PRESETS` / `EXPERIMENTAL_SUBAGENT_DEFAULTS`: Plan/Agent の反復シミュレーション用プリセット（実測較正が必要な暫定値）
   - `ASK_TURN_CACHE_RATIOS`: Ask の「何回目のやり取りか」→キャッシュ率の対応表
3. `npm run build` を実行
4. `dist/index.html` を再配布

> Plan/Agent はエージェントループの内部処理（反復回数・ツール呼び出し）が非公開のため、タスク規模プリセット × 反復シミュレーションで概算します。この係数は実測データ取得後に更新してください。

## 対応モデル

登録済みモデルすべて（provider別: OpenAI / Anthropic / Google / GitHub / Microsoft）が選択可能です。モデルの追加・削除は `src/rates.js` の `MODEL_RATES` のみで管理されるため、本READMEではモデル数・一覧を固定で記載しません。最新の一覧はツール内のモデル選択欄（provider別グルーピング）を参照してください。

> 単価は暫定値です。公式ドキュメントで最新値を確認のうえ `src/rates.js` を更新してください。

---

## ファイルサイズ推定の注意

ファイルサイズ(KB)からのトークン推定は簡易概算です。Word / Excel / PowerPoint / PDF は圧縮・画像・メタデータ・数式・表構造の影響が大きく、実際にCopilotへ投入されるトークン量とは差が出る可能性があります。精度を上げる場合は、対象本文をテキスト欄へ貼り付けてください。
